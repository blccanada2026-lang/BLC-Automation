# Staff Onboarding Rules ‚Äî BLC Nexus

## Overview
Staff onboarding is mandatory before any person can receive work or payroll. All designers, QC reviewers, TLs, and PMs must be onboarded first.

---

## Onboarding Workflow (required sequence)
1. CEO or ADMIN runs "Onboard Staff" from portal
2. Profile written to `DIM_STAFF_ROSTER`
3. Banking details written to `DIM_STAFF_BANKING`
4. CEO clicks "Generate Contract" ‚Üí contractor agreement Doc created in Google Drive
5. Contract metadata (URL, start date, jurisdiction) written to `DIM_STAFF_CONTRACTS`

---

## DIM_STAFF_ROSTER Required Columns
| Column | Purpose |
|---|---|
| `person_code` | Unique short code (e.g. DS1, TL1, RNR) |
| `supervisor_code` | person_code of TL who manages this person (blank for TLs+) |
| `pm_code` | person_code of the PM this person is mapped to |
| `pay_design` | Hourly rate for design work (in pay_currency) |
| `pay_qc` | Hourly rate for QC work (in pay_currency) |
| `pay_currency` | INR for designers, CAD for supervisors |
| `bonus_eligible` | TRUE/FALSE ‚Äî eligible for quarterly/annual bonus |

## DIM_STAFF_BANKING ‚Äî OFX Field Requirements by Country
| Bank Country | Required fields |
|---|---|
| **India** | `bank_name`, `bank_account_number`, `ifsc_code`, `account_type`, `beneficiary_name` |
| **Canada** | `bank_name`, `bank_account_number`, `institution_number`, `transit_number`, `beneficiary_name` |
| **USA** | `bank_name`, `bank_account_number`, `routing_number`, `account_type`, `beneficiary_name` |
| **International** | `bank_name`, `bank_account_number`, `swift_bic`, `iban` (if applicable), `beneficiary_name`, `bank_address` |

All records also store: `bank_country`, `bank_city`, `bank_address`, `ofx_recipient_id`

---

## DIM_STAFF_CONTRACTS ‚Äî Contract Generation Rules
- Generated by `StaffOnboarding.generateContract(actorEmail, personCode, options)` via `DocumentApp.create()`
- Saved to Google Drive folder: `BLC Contractor Agreements`
- Contract URL stored in `DIM_STAFF_CONTRACTS.doc_url`
- Contract type: always `INDEPENDENT_CONTRACTOR`
- Governing law: Saskatchewan, Canada (overrideable via `options.jurisdiction`)
- 12-clause agreement: Services, Compensation, Payment via OFX, IP Assignment, Confidentiality, Independent Contractor Status, Standards of Work, Term & Termination, Non-Solicitation, Liability Cap, Governing Law, General Provisions
- CEO only ‚Äî guarded by PAYROLL_RUN permission via RBAC

## StaffOnboarding Module (T8)
- File: `src/08-staff/StaffOnboarding.gs`
- WRITE_PERMISSIONS: `DIM_STAFF_ROSTER`, `DIM_STAFF_BANKING`, `DIM_STAFF_CONTRACTS`
- Public API: `onboardStaff()`, `generateContract()`, `getStaffList()`, `getBankingDetails()`
- `getBankingDetails()` ‚Äî CEO-only (PAYROLL_RUN permission)
- `getStaffList()` ‚Äî returns summary (no sensitive banking fields) for ADMIN+ roles
- Idempotent: re-running for existing person_code updates banking only; does not duplicate ROSTER row

---

## Bulk Staff Import (migration use)
- Sheet: `STG_STAFF_IMPORT` ‚Äî human-editable staging sheet with all DIM_STAFF_ROSTER + banking columns
- CEO fills rows, clicks "Bulk Import from Sheet" in portal
- Calls `StaffOnboarding.bulkOnboardStaff()`
- Status written back to `import_status` per row: `IMPORTED`, `SKIPPED_EXISTS`, or `ERROR: <message>`
- Rows already marked `IMPORTED` are skipped automatically ‚Äî safe to re-run after fixing errors
- Portal renders a result table showing each person's import status

---

## Portal ‚Äî Staff Management Panel
- Visible to CEO and ADMIN roles
- Table: person code, name, role, supervisor, PM, rates, banking status, contract status
- Actions per row: "Generate Contract", "View Banking" (CEO only)
- "Onboard Staff" form sections: Profile, Pay Config, Banking (OFX)
- Banking fields show/hide by `bank_country` selection (India / Canada / USA / International)
